Introducción

Este informe detalla la creación de objetos tipo VIEW para la gestión de la Clínica TodoSalud, enfocándose en la integridad referencial y la consolidación de datos distribuidos en tablas normalizadas.

Lógica de Consultas Implementada

  • JOINS Complejos: Unificación de tablas tmedico, tespecialidad y su tabla de intersección.
  • Funciones de Agregado: Uso de COUNT(*) para cuantificar especialidades.
  • Filtros de Grupo: Aplicación de HAVING para detectar médicos con múltiples especialidades.
  • Subconsultas de Exclusión: Uso de NOT IN para detectar brechas en el servicio médico.

Código SQL Destacado

Ejemplo de creación de vista para auditoría de calidad de datos:

CREATE OR REPLACE VIEW v_pacientes_sin_enfermedad AS
SELECT p.nombresp, p.paternop, p.maternop
FROM tpaciente p
JOIN tdiagnostico d ON p.codp = d.codp
WHERE d.codd NOT IN (SELECT codd FROM tenfermedadeneldiagnostico);

Conclusión

El diseño de estas soluciones permite a los ingenieros mejorar la toma de decisiones basada en datos estructurados, garantizando que la información fluya de manera lógica desde el código hasta la gerencia asistencial.
