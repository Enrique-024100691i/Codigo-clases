Introducción

Este proyecto detalla la creación de objetos de base de datos tipo Vista (VIEW) y consultas avanzadas aplicadas al sistema de la Clínica TodoSalud.

El objetivo principal es consolidar información distribuida en múltiples tablas normalizadas (Pacientes, Médicos, Diagnósticos y Especialidades) para facilitar la auditoría operativa y transformar datos crudos en información estratégica.

Estructura y Datos Iniciales

Se definieron 7 tablas principales con integridad referencial (Foreign Keys) para asegurar la consistencia del sistema médico.

Tablas Maestras

  • tpaciente: Datos personales de pacientes.
  • tmedico: Registro de personal médico.
  • tespecialidad: Catálogo de especialidades médicas.
  • tenfermedad: Catálogo de enfermedades.

Tablas Relacionales

  • tdiagnostico: Vincula pacientes con médicos.
  • tespecialidaddelmedico: Acreditación médica.
  • tenfermedadeneldiagnostico: Detalle de enfermedades detectadas.

Implementación de Vistas

1. Consolidación Médica (JOINS)

Para evitar consultas complejas recurrentes, creamos una vista que une médicos con sus respectivas especialidades.

-- Crear vista de médicos y sus especialidades
CREATE VIEW v_medicos_especialidades AS
SELECT m.*, e.nombreesp, e.descripcionesp, em.fechaobtencion
FROM tmedico m
JOIN tespecialidaddelmedico em ON m.codm = em.codm
JOIN tespecialidad e ON em.codesp = e.codesp;

2. Análisis de Alta Especialización

Utilizando la cláusula HAVING sobre la vista anterior para identificar médicos con 2 o más especialidades.

SELECT nombresm, paternom, maternom, COUNT(*) AS cantidad
FROM v_medicos_especialidades
GROUP BY nombresm, paternom, maternom
HAVING COUNT(*) >= 2;

3. Detección de Brechas y Registros Huérfanos

Mediante subconsultas de exclusión (NOT IN), identificamos especialidades sin cobertura y enfermedades no detectadas aún.

-- Especialidades sin médico asociado
CREATE OR REPLACE VIEW v_especialidades_sin_medico AS
SELECT * FROM tespecialidad
WHERE codesp NOT IN (SELECT codesp FROM tespecialidaddelmedico);

Conclusiones

  • La implementación de Vistas optimiza el rendimiento del equipo de desarrollo al simplificar el acceso a datos complejos.
  • El uso de NOT IN permite realizar auditorías de calidad de datos, detectando diagnósticos incompletos o falta de cobertura médica.
  • Estas herramientas transforman una base de datos operativa en un activo estratégico para la toma de decisiones en salud.
