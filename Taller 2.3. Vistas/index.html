1. Introducción

Este proyecto detalla la implementación de objetos de base de datos tipo Vista (VIEW) y consultas avanzadas aplicadas a la Clínica TodoSalud. El enfoque principal es la transformación de datos crudos en información estratégica mediante el uso de uniones (JOINS), subconsultas de exclusión y funciones de agregado. [cite: 20, 24]

2. Estructura de Datos

La base de datos se fundamenta en la integridad referencial entre las siguientes entidades:

  • tpaciente: Registro de usuarios de la clínica.
  • tmedico: Personal médico y datos de contacto.
  • tespecialidad: Catálogo de ramas médicas disponibles.
  • tdiagnostico: Encuentros clínicos entre médicos y pacientes.

3. Desarrollo y Lógica SQL

Paso 5.1 & 5.2: Consolidación y Conteo de Especialidades

Se crea la vista v_medicos_especialidades uniendo tres tablas para visualizar qué especialidad ostenta cada médico. [cite: 29, 30]

CREATE VIEW v_medicos_especialidades AS
SELECT m.*, e.nombreesp, em.fechaobtencion
FROM tmedico m
JOIN tespecialidaddelmedico em ON m.codm = em.codm
JOIN tespecialidad e ON em.codesp = e.codesp;

Posteriormente, se utiliza COUNT(*) y GROUP BY para determinar la carga de especialización por profesional. [cite: 42]

Paso 5.4: Detección de Brechas (Enfermedades no detectadas)

Utilizando la lógica de exclusión con NOT IN, identificamos patologías que no han sido asignadas en ningún diagnóstico actual. [cite: 67, 74]

CREATE OR REPLACE VIEW v_enfermedades_no_detectadas AS
SELECT * FROM tenfermedad
WHERE code NOT IN (SELECT code FROM tenfermedadeneldiagnostico);

Paso 5.7: Auditoría de Calidad (Diagnósticos Incompletos)

Esta consulta detecta errores humanos: pacientes con una cita médica registrada pero sin una enfermedad formalmente vinculada en el sistema. [cite: 100, 101]

CREATE OR REPLACE VIEW v_pacientes_sin_enfermedad AS
SELECT p.nombresp, p.paternop, p.maternop
FROM tpaciente p
JOIN tdiagnostico d ON p.codp = d.codp
WHERE d.codd NOT IN (SELECT codd FROM tenfermedadeneldiagnostico);

4. Conclusión

La implementación de estas vistas permite a la clínica automatizar auditorías operativas y mejorar la toma de decisiones asistenciales, asegurando que ningún paciente quede con registros incompletos y optimizando la oferta médica. [cite: 116, 117]
