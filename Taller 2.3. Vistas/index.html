1. Introducción del Proyecto

Este proyecto se centra en la gestión de datos de la Clínica TodoSalud utilizando Oracle SQL. El objetivo es transformar datos crudos en información estratégica mediante el uso de VIEWS (Vistas), JOINS y subconsultas complejas.

Objetivo: Fortalecer la capacidad analítica mediante el dominio de la integridad referencial y la consolidación de tablas normalizadas como Pacientes, Médicos, Especialidades y Diagnósticos.

2. Modelo de Datos (Esquema)

El sistema se basa en 7 tablas principales interconectadas:

  • tpaciente: Datos personales de los pacientes.
  • tmedico: Registro del cuerpo médico.
  • tespecialidad: Catálogo de especialidades médicas.
  • tdiagnostico: Encuentros clínicos entre médicos y pacientes.
  • tespecialidaddelmedico: Tabla de intersección (Médico-Especialidad).
  • tenfermedad: Catálogo de patologías.
  • tenfermedadeneldiagnostico: Relación entre diagnósticos y enfermedades.

3. Implementación SQL Destacada

A. Consolidación de Información Médica

Se creó la vista v_medicos_especialidades para unificar la información del médico con su nombre de especialidad textual, eliminando la necesidad de escribir JOINS repetitivos.

CREATE VIEW v_medicos_especialidades AS
SELECT m.*, e.nombreesp, e.descripcionesp, em.fechaobtencion
FROM tmedico m
JOIN tespecialidaddelmedico em ON m.codm = em.codm
JOIN tespecialidad e ON em.codesp = e.codesp;

B. Identificación de Brechas (Registros Huérfanos)

Una funcionalidad crítica es detectar qué enfermedades no se han diagnosticado aún o qué especialidades no tienen médicos asignados. Esto se logra con operadores NOT IN:

CREATE OR REPLACE VIEW v_especialidades_sin_medico AS
SELECT * FROM tespecialidad
WHERE codesp NOT IN (SELECT codesp FROM tespecialidaddelmedico);

C. Auditoría de Calidad

La vista v_pacientes_sin_enfermedad identifica errores de registro donde existe un diagnóstico pero no se vinculó formalmente a una enfermedad, permitiendo corregir omisiones administrativas.
