Modelo Relacional

El diseño se basa en una estructura centralizada en el Diagnóstico, el cual vincula al Paciente con el Médico. Además, se implementa una relación de muchos a muchos entre Médicos y Especialidades, y entre Diagnósticos y Enfermedades.

Entidades Principales

  • • tpaciente: Datos personales y demográficos.
  • • tmedico: Información profesional de contacto.
  • • tenfermedad: Catálogo de patologías.

Entidades de Negocio

  • • tespecialidad: Áreas médicas disponibles.
  • • tdiagnostico: Registro de la consulta médica.

Tablas Relacionales

  • • tespecialidaddelmedico: Asigna especialidades a médicos.
  • • tenfermedadeneldiagnostico: Detalla enfermedades por diagnóstico.

Script de Creación e Inserción

clinica_schema.sql 
-- ELIMINAR TABLAS EXISTENTES
drop table tenfermedadeneldiagnostico cascade constraints;
drop table tespecialidaddelmedico cascade constraints;
drop table tdiagnostico cascade constraints;
drop table tenfermedad cascade constraints;
drop table tespecialidad cascade constraints;
drop table tmedico cascade constraints;
drop table tpaciente cascade constraints;

-- CREACIÓN DE TABLAS
create table tpaciente (
    codp varchar2(4) primary key,
    tipodocp varchar2(15),
    nrodocp varchar2(12),
    paternop varchar2(50),
    maternop varchar2(50),
    nombresp varchar2(50),
    fechanacimientop date,
    generop char(1)
);

create table tmedico (
    codm varchar2(4) primary key,
    tipodocm varchar2(15),
    nrodocm varchar2(12),
    paternom varchar2(50),
    maternom varchar2(50),
    nombresm varchar2(50),
    celularm varchar2(12),
    fechanacimientom date,
    generom char(1)
);

create table tdiagnostico (
    codd varchar2(4) primary key,
    fechahorad timestamp,
    descripciond varchar2(100),
    codp varchar2(4),
    codm varchar2(4),
    foreign key (codp) references tpaciente(codp),
    foreign key (codm) references tmedico(codm)
);

Análisis y Reportes (Vistas)

1 & 2. Gestión de Médicos y Especialidades

Integración de datos maestros de médicos con sus certificaciones para análisis de carga académica.

create view v_medicos_especialidades as
select m.*, e.nombreesp, e.descripcionesp, em.fechaobtencion
from tmedico m
join tespecialidaddelmedico em on m.codm = em.codm
join tespecialidad e on em.codesp = e.codesp;

-- Conteo por médico
select nombresm, count(*) as cantidad_especialidades
from v_medicos_especialidades
group by nombresm, paternom, maternom;

4 & 5. Auditoría de Datos (Elementos sin Uso)

Identificación de enfermedades no diagnosticadas y especialidades sin staff médico asignado utilizando subconsultas NOT IN.

create or replace view v_enfermedades_no_detectadas as
select * from tenfermedad
where code not in (select code from tenfermedadeneldiagnostico);

create or replace view v_especialidades_sin_medico as
select * from tespecialidad
where codesp not in (select codesp from tespecialidaddelmedico);
