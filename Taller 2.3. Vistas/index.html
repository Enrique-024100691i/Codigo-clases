Introducción

El taller se enfoca en la transformación de datos crudos en información estructurada de alto valor clínico. Se utilizó Oracle SQL Developer para manejar la integridad referencial y consolidar datos de múltiples tablas mediante Vistas (VIEW).

Desarrollo y Lógica SQL

01

Consolidación Médica

Vistas y Triple Joins

Se unieron las tablas de Médicos, Especialidades y su relación para evitar consultas complejas repetitivas.

CREATE VIEW v_medicos_especialidades AS
SELECT m.*, e.nombreesp
FROM tmedico m
JOIN tespecialidaddelmedico em ON m.codm = em.codm
JOIN tespecialidad e ON em.codesp = e.codesp;
02

Alta Especialización

Group By & Having

Identificación de médicos con múltiples especialidades (conteo >= 2) para análisis de capacidad.

SELECT nombresm, COUNT(*) as cant
FROM v_medicos_especialidades
GROUP BY nombresm, paternom, maternom
HAVING COUNT(*) >= 2;
03

Identificación de Brechas

Subconsultas y Exclusión

Uso de NOT IN para detectar enfermedades que nunca han sido diagnosticadas o especialidades sin personal.

SELECT * FROM tenfermedad
WHERE code NOT IN (
  SELECT code FROM
  tenfermedadeneldiagnostico
);
