Implementación de Base de Datos Relacional en Oracle SQL

Descripción del Proyecto

Este script automatiza la creación, poblamiento y consulta de una base de datos para una clínica. Gestiona la relación entre pacientes, médicos, especialidades y diagnósticos, permitiendo identificar patrones de salud y especialización del personal médico.

Script SQL Completo

script_clinica.sql 
-- ==========================================================
-- PROYECTO: SISTEMA DE CLÍNICA
-- AUTOR: Zavaleta Fuentes Paolo Enrique
-- ==========================================================

-- 1. LIMPIEZA DE ENTORNO (DDL)
drop table tenfermedadeneldiagnostico cascade constraints;
drop table tespecialidaddelmedico cascade constraints;
drop table tdiagnostico cascade constraints;
drop table tenfermedad cascade constraints;
drop table tespecialidad cascade constraints;
drop table tmedico cascade constraints;
drop table tpaciente cascade constraints;

-- 2. CREACIÓN DE TABLAS
create table tpaciente (
    codp varchar2(4) primary key,
    tipodocp varchar2(15),
    nrodocp varchar2(12),
    paternop varchar2(50),
    maternop varchar2(50),
    nombresp varchar2(50),
    fechanacimientop date,
    generop char(1)
);

-- [Resto del código de creación...]
create table tmedico (
    codm varchar2(4) primary key,
    tipodocm varchar2(15),
    nrodocm varchar2(12),
    paternom varchar2(50),
    maternom varchar2(50),
    nombresm varchar2(50),
    celularm varchar2(12),
    fechanacimientom date,
    generom char(1)
);

-- [Inserción de datos...]
insert into tpaciente values ('p01','dni','11111111','salas','rivas','juan',date '2000-02-02','m');
insert into tmedico values ('m1','dni','33333333','zela','ramirez','javier','999999999',date '1970-10-10','m');

-- 3. VISTAS Y REPORTES AVANZADOS
-- Vista para médicos y sus especialidades
create view v_medicos_especialidades as
select m.*, e.nombreesp, e.descripcionesp, em.fechaobtencion
from tmedico m
join tespecialidaddelmedico em on m.codm = em.codm
join tespecialidad e on em.codesp = e.codesp;

-- Consulta: Médicos con 2 o más especialidades
select nombresm, count(*) as cantidad
from v_medicos_especialidades
group by nombresm, paternom, maternom
having count(*) >= 2;

Análisis de Consultas

Vistas Dinámicas

Se implementaron vistas para simplificar JOINs complejos, permitiendo reportes rápidos sobre la disponibilidad de médicos por especialidad.

Integridad Referencial

Uso de CASCADE CONSTRAINTS para asegurar que el despliegue del script sea limpio y sin conflictos de llaves foráneas.
